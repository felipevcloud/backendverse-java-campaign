## Planejamento

1. Qual classe representa o aprendiz? - AprendizRegistro
2. Qual classe executa o programa? - RegistroAcademia
3. Quais dados cada aprendiz precisa ter? - nome, tecnica principal, nivel, status
4. Quantos aprendizes serão criados? - 3 aprendizes
5. Qual posição do array ficará vazia para testar null? - 2
6. Como o programa vai evitar erro ao encontrar null? - If/Else

## Descrição da missão
O sistema registra aprendizes da Academia Backend em um array, percorre os registros, identifica slots vazios com null e demonstra como duas referências podem apontar para o mesmo objeto.

## Como executar
1. Abrir o projeto no IntelliJ.
2. Executar a classe `RegistroAcademia`.
3. Verificar a saída no console.

## Resultado esperado
O console deve mostrar os aprendizes registrados, identificar pelo menos um slot vazio e demonstrar alteração em um objeto por referência compartilhada.
